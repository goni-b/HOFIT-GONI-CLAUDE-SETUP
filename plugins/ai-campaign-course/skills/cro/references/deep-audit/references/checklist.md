# Landing Page Audit Checklist

A 100+ point checklist organized by section. Use as a printable reference. Mark each item as **✅ Strong / 🟡 Acceptable / 🔴 Weak / ❌ Missing**.

---

## A. Above the fold (the first 3 seconds)

- [ ] Headline answers "what's in it for me?" within 5 seconds of arrival
- [ ] Headline names a specific outcome (not a vague benefit)
- [ ] Headline clarifies *who it's for* (specific audience)
- [ ] Sub-headline expands the promise with a tangible specific
- [ ] Hero visual reinforces the offer (not generic stock, not decorative)
- [ ] Primary CTA is visible without scrolling on every common viewport
- [ ] Page-to-source message match (ad headline ≈ page headline)
- [ ] No autoplay video that delays the message
- [ ] Logo present but not dominant
- [ ] Trust marker visible above the fold (rating, customer count, big-name logo, or press mention)

## B. Value proposition

- [ ] Quantified benefit ("Cut payroll time by 4 hours/week" beats "Save time")
- [ ] Differentiator from the obvious alternatives clear (including doing nothing)
- [ ] Reading level around 8th grade
- [ ] Sentences under 20 words
- [ ] Active voice (not "is provided by us" but "you get")
- [ ] No jargon that requires industry knowledge to parse
- [ ] Outcome described in customer language, not internal language
- [ ] Page title (browser tab) matches the hero headline

## C. Body / story flow

- [ ] Logical progression: problem → solution → proof → action
- [ ] One idea per section — each section has a single job
- [ ] Subheadings tell the story by themselves (the skim test)
- [ ] White space breaks up dense text
- [ ] Bullet lists used for benefits, paragraphs for stories/explanation
- [ ] Visual hierarchy: most important = largest/highest contrast
- [ ] No "false bottoms" (sections that look like the page ends but doesn't)
- [ ] Image-heavy sections balanced with text-heavy ones

## D. Call-to-action

- [ ] Verb-first, outcome-focused copy ("Get my free audit" not "Submit")
- [ ] High-contrast button color vs. page background
- [ ] Ample whitespace around button
- [ ] Repeated at logical intervals on long pages
- [ ] Mobile tap target ≥48×48 px
- [ ] Sticky bottom CTA on long mobile pages
- [ ] Click-trigger micro-copy near button ("No credit card. Cancel anytime.")
- [ ] Button doesn't compete with other buttons in visual weight
- [ ] Hover/tap states clear (button looks clickable)
- [ ] Primary CTA distinct from secondary CTA (one is unmistakably the main action)

## E. Forms

- [ ] Field count is minimum to qualify (every extra field hurts)
- [ ] Labels above fields (not inside as disappearing placeholders)
- [ ] Labels visible at all times after typing starts
- [ ] Real-time error messages (not on submit only)
- [ ] Optional fields clearly marked
- [ ] Required fields not marked with asterisks alone (use word "required")
- [ ] Privacy reassurance near email field
- [ ] One column layout (not two)
- [ ] Correct keyboard types on mobile (email, tel, number)
- [ ] Autofill works (correct `autocomplete` HTML attributes)
- [ ] Progress indicator on multi-step forms
- [ ] Submit button disabled until valid (or clear validation feedback)
- [ ] Success state after submit (not just a redirect)
- [ ] Smart defaults pre-selected where possible
- [ ] No CAPTCHA unless absolutely necessary (tanks conversion)

## F. Trust & social proof

- [ ] Testimonials with full name, photo, role, company (no anonymous)
- [ ] Specific outcome quotes ("Increased MQLs 47% in 6 weeks")
- [ ] Customer logo bar
- [ ] Press mentions / "as seen in"
- [ ] Numerical proof ("12,000 teams", "$847M processed")
- [ ] Reviews/ratings with recency
- [ ] Trust badges near friction points (security badges near payment)
- [ ] Money-back guarantee or free trial clearly stated
- [ ] Real photos of real people (not stock)
- [ ] Founder photo + brief story builds liking
- [ ] Privacy policy linked (not buried)
- [ ] Contact info visible (phone, address, real email)
- [ ] HTTPS / lock icon visible
- [ ] No broken testimonial photos

## G. Objections & FAQ

- [ ] Top 5-10 objections addressed somewhere on page
- [ ] FAQ section near the bottom for "considering" buyers
- [ ] Comparison to alternatives if relevant (comparison table)
- [ ] Pricing transparent (no "contact us for pricing" unless enterprise)
- [ ] What's NOT included clarified for honesty
- [ ] Refund/cancellation policy stated
- [ ] Time-to-value clear ("setup in 5 minutes", "first results in 2 weeks")

## H. Urgency & scarcity

- [ ] Implicit urgency (the pain is ongoing) is clearly conveyed
- [ ] Explicit urgency (deadline, cohort, limited stock) is real, not manufactured
- [ ] Countdown timers, if used, are honest
- [ ] No fake "X people viewing this now" theater
- [ ] Bonus or discount has a clear, real expiration

## I. Page speed & technical

- [ ] Loads in <3s on 4G mobile (LCP <2.5s ideal)
- [ ] First Contentful Paint <1.8s
- [ ] Cumulative Layout Shift <0.1
- [ ] Time to Interactive <3.8s
- [ ] Images compressed (WebP/AVIF where possible)
- [ ] Images sized to viewport (no 4000×3000 image displayed at 400×300)
- [ ] Lazy-loading on below-fold images
- [ ] Critical CSS inline, rest deferred
- [ ] No render-blocking JavaScript above-fold
- [ ] CDN serving static assets (Cloudflare, BunnyCDN, etc.)
- [ ] Browser caching enabled for repeat visitors
- [ ] No console errors
- [ ] No 404s on linked resources
- [ ] Forms work in Safari iOS *and* Chrome Android (test, don't assume)

## J. Mobile experience

- [ ] No horizontal scroll on any common viewport
- [ ] Tap targets ≥48×48 px with 8 dp spacing
- [ ] Font size ≥16 px for body (no zooming required)
- [ ] Sticky CTA on long pages
- [ ] No popups that cover the screen on mobile (Google penalizes them)
- [ ] Hero image cropped/optimized for portrait viewports
- [ ] Forms autofill correctly
- [ ] Email links use `mailto:`, phone links use `tel:`
- [ ] No hover-only interactions (mobile has no hover)

## K. Distractions removed

- [ ] No global navigation menu on dedicated paid landing pages
- [ ] No outbound links above the fold (besides logo home link)
- [ ] No social media icons that lead away from page
- [ ] No carousel/slider in hero (data shows they hurt conversions)
- [ ] No exit-intent popup that fires too early
- [ ] No autoplay video with sound
- [ ] No competing CTAs in close proximity
- [ ] No unnecessary forms (just the one for the goal)
- [ ] No live chat that interrupts within first 30 seconds

## L. Analytics & tracking (for ongoing optimization)

- [ ] GA4 installed and firing on all pages
- [ ] Conversion events tracked (form submit, button click, video play, etc.)
- [ ] UTM parameters correctly captured from ad sources
- [ ] Heatmap tool installed (Hotjar, Microsoft Clarity)
- [ ] Session recording capturing samples
- [ ] Form analytics (Hotjar Forms, Formisimo, etc.)
- [ ] Funnel report set up in GA4
- [ ] A/B testing tool ready (VWO, Optimizely, Convert, Google Optimize replacement)

## M. Mistakes that kill conversions (every one is a 🔴 if present)

- [ ] Hero says "Welcome to [Company]" or similar empty greeting
- [ ] Hero is generic stock photo of smiling diverse coworkers in an office
- [ ] CTA says "Submit", "Click here", or "Learn more"
- [ ] Form asks for phone number when email would do
- [ ] Multiple competing CTAs above the fold
- [ ] Pricing hidden behind "contact sales"
- [ ] Testimonial with no name or just "John D."
- [ ] Page tries to be everything to everyone
- [ ] Carousels that auto-advance
- [ ] Pop-up immediately on page load
- [ ] Cookie consent that blocks the page
- [ ] Lorem ipsum in production (yes, this happens)
- [ ] "We" / "Our" outweighs "You" / "Your" 2:1 or worse
- [ ] Page is one giant block of text with no breaks

---

## Severity rating (after marking)

**🔴 Critical (severity 4)** — affects most visitors, high revenue impact. Fix this week.
**🟡 Major (severity 3)** — affects many visitors, real revenue impact. Test or fix this sprint.
**🟢 Minor (severity 2)** — cosmetic or low-traffic issue. Backlog.
**⚪ Polish (severity 1)** — nice-to-have. Backlog.
**(no rating)** — strong already.

Rule: don't deliver an audit longer than the user can act on. Top 5-10 items prioritized by ICE beats 47 items in a spreadsheet.
